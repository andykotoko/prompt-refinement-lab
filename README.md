# Prompt Refinement Lab

一个用于记录与实践 **提示词工程（Prompt Engineering）** 的实验仓库。

本仓库关注的不是“写出最强提示词”，而是通过 **统一命名、可复现结构与测试语境**，系统性探索提示词在不同任务中的稳定性、可迁移性与工程化路径。

内容以中文为主，所有文件均为 **可直接使用的提示词（prompt）**。

---

## 仓库动机

在正式进入科研或长期研究之前，我希望通过一个 **低风险、可迭代的实验仓库**，训练如下能力：

- 将模糊的使用经验转化为 **可执行的提示词结构**
- 对提示词进行 **版本化管理与对照测试**
- 区分「共享系统指令」与「任务型提示词」在实验中的角色
- 建立可复用的提示词改进流程，而非一次性结果

因此建立了 Prompt Refinement Lab。

## 提示词说明

### encyclopedic_system_prompt.txt

百科全书级系统提示词，用作 **共享的系统指令（system prompt）**：

- 不包含具体测试问题
- 用于定义角色、方法论、沟通风格与价值取向
- 可作为其他任务型提示词的公共背景

---

### camera-prepost.prompt

聚焦 **相机前期 / 后期** 场景的提示词：

- 拍摄参数、构图、光线等前期决策
- 后期处理、风格选择与流程拆解
- 通过测试问题验证提示词在复杂场景下的稳定性

---

### lexical-writing.prompt

聚焦英语语言层面的提示词：

- 英语词根与词源分析
- 同义词区分与语义精度
- 写作中词汇选择与表达风格控制

该提示词用于分析 **词汇层选择如何影响整体写作质量**。

---

### cpp-evaluation.prompt

面向编程与代码生成场景的提示词：

- C / C++（及部分 Python）代码生成
- 不同提示词对代码结构、可读性与正确性的影响
- 通过对照测试分析提示词的稳定性

---

### relationship-analysis.prompt

聚焦人际关系与沟通场景的提示词：

- 情境分析与信息抽取
- 回应策略与表达方式生成
- 不同表达路径下的效果对比

---

## 提示词改进的一般流程

以下流程并非固定模板，而是在多轮实践中逐步形成：

1. **确定稳定的初始提示词（baseline）**
2. **明确问题，再针对性修改提示词**
3. **引入外部信息源，避免闭门调参**
4. **构造测试问题或对照场景进行验证**
5. **记录输出结果并进行复盘**
6. **进入下一轮迭代**

---

## 关于测试与 benchmark

本仓库中的测试更偏向 **压力测试与结构验证**，而非高频日常使用：

- 测试问题往往刻意设置为复杂或边缘情境
- 用于观察提示词在高不确定性条件下的表现

最终优化目标始终以 **真实使用场景的稳定性** 为准。
